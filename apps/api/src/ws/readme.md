# ws

## online

| 状态 | 含义 | 如何触发 | 对外可见性 | 需要推送给谁 |
| --- | --- | --- | --- | --- |
| online | 在线(活跃) | 连接成功+已登录 | 公开 | 好友+共同社区成员 |
| idle | 离开\afk | 10分钟内无任何操作(鼠标\键盘\触摸)\用户手动设置 | 公开 | 同上 |
| dnd | 勿扰 | 用户手动设置 | 公开 | 同上 |
| invisible | 隐身 | 用户手动设置 | 显示为 offline | 服务器知道真实状态, 但不广播 |
| offline | 离线 | 连接断开\连接超时\退出登录 | 公开 | 同上 |

注意
- 每个用户可以存在多个 socket（多设备/标签页）。我们在 Redis 中维护 `user:sockets:${userId}` 计数，所有连接都加入 `user:${userId}` 房间。
- 批量获取：客户端可发送 `status:batch` 事件带 id 列表，服务器通过 Redis pipeline 返回映射。
- 未登录用户仍允许连接并加入评论/频道房间，但不会在存在状态表中出现，也无权设置状态。
- afk: away from keyboard (离开键盘)
- presence: 在线状态, 在场状态
